# 🎰 Giveaway Bot

Telegram-бот для организации розыгрышей и слотовых лотерей.

## Стек
- Python 3.11+
- aiogram 3.x
- aiosqlite (SQLite)
- aiogram FSM

## Структура проекта

```
giveaway_bot/
├── bot.py                    # Точка входа
├── config.py                 # Конфигурация
├── requirements.txt
├── .env.example
│
├── db/
│   └── database.py           # Инициализация БД и все запросы
│
├── handlers/
│   ├── start.py              # /start, главное меню, управление каналами
│   ├── classic.py            # FSM создания классического розыгрыша
│   ├── slots.py              # FSM создания лотереи по слотам
│   ├── my_projects.py        # Просмотр, подведение итогов, закрытие
│   ├── participation.py      # Deep link, проверка подписок, бронь слотов
│   └── payments.py           # pre_checkout_query, successful_payment
│
├── keyboards/
│   └── keyboards.py          # Все inline-клавиатуры
│
├── middlewares/
│   └── admin.py              # Авто-регистрация пользователей
│
└── utils/
    ├── states.py             # FSM-состояния
    └── helpers.py            # Вспомогательные функции
```

## Запуск

```bash
# 1. Установить зависимости
pip install -r requirements.txt

# 2. Настроить переменные окружения
cp .env.example .env
# Отредактировать .env

# 3. Запустить
BOT_TOKEN=xxx python bot.py
```

## Переменные окружения

| Переменная | Описание |
|---|---|
| `BOT_TOKEN` | Токен бота от @BotFather |
| `ADMIN_IDS` | ID администраторов через запятую (опционально) |
| `DB_PATH` | Путь к SQLite файлу (по умолчанию `giveaway.db`) |

## Логика розыгрышей

### Классический розыгрыш
1. Создатель выбирает канал → вводит текст/фото → условие завершения (время/кол-во) → число победителей → текст кнопки → спонсоры
2. Публикуется пост с кнопкой → Deep Link ведёт в бота
3. Участник: проверка подписок → регистрация → уведомление
4. Подведение итогов: рандомный выбор победителей → обновление поста → уведомление победителей

### Лотерея по слотам
1. Создатель: канал → текст/фото → кол-во слотов → тип оплаты → спонсоры
2. Winning slot выбирается случайно и сохраняется в БД (тайно)
3. Публикуется пост с сеткой кнопок-слотов
4. Участник: клик на слот → Deep Link → проверка подписок → оплата (если платно) → бронирование
5. После брони: проверяется совпадение с winning_slot → мгновенная победа или "ждите"
6. При победе: обновляется пост, уведомляются победитель и создатель

## Оплата (Telegram Stars)
- Используется `bot.send_invoice` с `currency="XTR"` и пустым `provider_token`
- При коллизии (слот занят после оплаты) — автоматический возврат через `bot.refund_star_payment`
